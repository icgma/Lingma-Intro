委派任务给 AI，让编程智能体自主规划、执行和验证，端到端完成开发任务。

Editor 模式 vs Quest 模式

Lingma IDE 提供两种工作模式，可通过左上角按钮随时切换。

✏️ Editor 模式

实时协作编程

  • 你主导编码，AI 实时辅助
  • 代码补全、行内对话、智能修复
  • 适合日常编码和小型修改
  • 快捷键：Ctrl+I 行内对话
🎯 Quest 模式

任务委派，自主执行

  • AI 主导执行，你审查确认
  • 自主规划、编辑代码、运行命令
  • 适合复杂功能开发和项目搭建
  • 快捷键：Ctrl+E 进入

三种任务场景

创建 Quest 任务时，可选择以下场景。不选则 AI 自动判断最适合的方式。

📋

Spec 驱动

适用于复杂功能开发、重构或需要严格质量把控的项目。

📝

工作流程：AI 先与你对齐需求范围 → 设计实施方案和验收标准（生成 Spec 文档）→ 你确认后开始执行 → 按 Spec 逐项完成。

# 适合 Spec 驱动的任务： "为我的 Flask 应用添加用户认证模块， 包含注册、登录、密码重置功能， 使用 JWT Token，密码用 bcrypt 加密"
🌐

搭建网站

适用于从零创建网站或快速原型。描述需求后 AI 自动搭建页面和整体结构。

# 适合搭建网站的任务： "帮我创建一个个人博客网站， 深色主题，包含首页、文章列表和关于页面， 使用 HTML/CSS/JS，支持响应式设计"
💡

该模式支持实时预览——在产物区的 Preview Tab 可以一边开发一边看效果。

🧪

原型探索

适用于快速验证想法或创意实验。AI 将一个想法转化为可运行的原型。

# 适合原型探索的任务： "做一个贪吃蛇小游戏， 用 Python + Pygame， 支持键盘操控和计分功能"

执行环境

Quest 任务支持两种执行环境，影响代码变更的位置和隔离程度。

💻

Local（本地）

直接在主工作区修改。启动成本为零，适合简单任务和快速验证。支持多个本地任务并行。

隔离性低 — 直接改主代码
启动速度即时
适用场景小修改、快速验证
🌿

Worktree（并行）

后台创建隐藏工作区。主分支保持干净，适合中等复杂任务和多任务并行。可无限次迭代。

隔离性高 — 独立工作区
启动速度稍慢（需创建）
适用场景复杂功能、并行开发

Quest 工作流

从创建任务到完成交付的完整流程。

1

创建任务

按 Ctrl+E 进入 Quest 模式，点击 "新建 Quest" 按钮，选择场景和执行环境。

2

描述需求

用自然语言清晰描述你的开发需求。AI 可能会先提出澄清问题以确认细节。

3

AI 规划执行

AI 生成执行计划（To-do List），逐步完成每个步骤。你可以在对话区实时查看进度。

4

审查变更

在产物区的 Changed Files 标签查看所有代码变更。可以逐文件审查、接受或丢弃。

5

追加或调整

如果结果不满意，可以直接在输入框追加新需求，AI 会调整计划继续工作。也可点击 "暂停" 按钮暂停任务。

📊 任务状态说明

状态图标含义
Running🔄任务正在执行中
Action Required⏳等待你的确认或操作
Ready✅任务已完成
Error❌执行过程中遇到错误

使用技巧

让 Quest 模式更高效的实用建议。

✍️

写好任务描述

描述越具体越好。包含技术选型、功能需求、UI 要求等细节，AI 的执行质量越高。

📐

配合规则使用

在设置中配置编码规则，Quest 任务会自动遵循，确保代码风格一致。了解规则 →

🔌

搭配 MCP 服务

配置 MCP 后，Quest 智能体可以调用外部工具（查文档、查数据库等），能力更强。了解 MCP →

🌿

复杂任务用 Worktree

大功能开发建议使用 Worktree 环境，保持主分支干净，确认满意后再合并。

🔄

迭代而非重做

如果结果不完美，追加修改需求比重新创建任务更高效。AI 会在已有基础上优化。

📁

多任务并行

可以同时运行多个 Quest 任务，如一个做前端一个做后端。任务列表按最后交互时间排序。
