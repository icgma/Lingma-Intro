编程智能体（Agent）

智能体是灵码最强大的功能——不只是回答问题，而是帮你"做事"。

🧠 通俗理解

AI 助手 vs 编程智能体

普通 AI 助手就像一本会说话的教材——你问它"这个函数怎么用"，它能回答。但你需要自己动手写代码、自己创建文件、自己安装软件包。

而编程智能体更像一个全能实习生——你告诉它"帮我做一个数据分析报告"，它会：自己想好步骤、自己创建文件、自己写代码、自己安装缺少的库、自己运行测试。你只需要说"做什么"，不需要说"怎么做"。

🔧 智能体能做什么

📋

任务拆解

你说"做一个网站"，它会自动拆成：① 创建文件结构 ② 写 HTML ③ 写 CSS ④ 添加交互功能 ⑤ 测试。

比喻：就像项目经理把大任务分解为小任务。

🔍

环境感知

它会自动扫描你的项目：用了什么语言？有哪些文件？用了什么框架？然后根据这些信息给出匹配的代码。

比喻：就像新同事上班第一天先熟悉公司环境。

📝

多文件编辑

可以同时创建和修改多个文件。比如一次性创建 HTML、CSS、JavaScript 三个文件并让它们配合工作。

比喻：就像同时编辑文章的正文、目录和参考文献。

💻

运行终端命令

自动运行命令，比如安装库（pip install pandas）、运行脚本（python main.py）。

比喻：就像实习生不但写了文档，还帮你排版打印了。

🔌

MCP 工具调用

可以使用你安装的 MCP 工具：搜索网页、查数据库、读文档。 了解 MCP →

比喻：实习生不但能干活，还有"人脉"（外部工具）。

🧠

能记住你的偏好

它知道你喜欢什么代码风格、常用什么库、项目有什么特点。用得越多，越懂你。

比喻：就像用久了的微信输入法，会推荐你常用的词。

🚀 如何使用智能体

1

进入智能体模式

按 Ctrl+E 进入 Quest 模式——这就是智能体的主要工作界面。也可以在 AI 对话框左下角找到模式切换按钮，选择"Agent"模式。

2

用大白话描述任务

不需要技术术语！直接用中文说你想要什么：

# ✅ 好的描述 帮我做一个 Python 程序，功能是： 读取 scores.csv 里面同学们的成绩， 计算每个人的平均分和总分， 然后生成一个新的 CSV 文件叫 results.csv # ✅ 也很好（可以更简单） 帮我做个网页，展示班级活动照片， 九宫格布局，鼠标悬停放大图片
3

审查并确认

智能体会把工作结果展示给你。你可以：

  • ✅ 接受——满意，应用修改
  • 💬 追加需求——"颜色改成蓝色""再加一个按钮"
  • 🗑️ 丢弃——不满意，放弃这次修改
  • 📖 请求解释——"帮我逐行解释这段代码"

技能（Skills）

技能是给 AI 装的"专业技能包"——让它在某个领域更专业。

🧠 通俗理解

什么是技能？

想象 AI 是一个通才员工——什么都能做一点，但都不够专精。技能就像送他去上了一个培训班：上完"代码审查培训班"后，他就能更专业地检查你的代码质量了。

技术上说，技能就是一个文件夹，里面有一份 SKILL.md 文件（技能说明书），告诉 AI 在遇到特定任务时应该怎么处理。

📁 技能长什么样？

一个技能就是一个文件夹，最简单的情况下只需要一个 SKILL.md 文件：

my-skill/ # 技能文件夹 ├── SKILL.md # 必需：技能说明书 ├── scripts/ # 可选：辅助脚本 ├── examples/ # 可选：参考示例 └── resources/ # 可选：其他资源

📍 技能存在哪里？

👤

用户级技能

存放在 ~/.lingma/skills/。你个人所有项目都能用。

📂

项目级技能

存放在项目的 .lingma/skills/。仅当前项目可用，适合团队共享。

⚡ 技能怎么触发？

有两种方式让 AI 使用技能：

🤖 自动触发

AI 自己判断

当你的对话内容和某个技能的描述匹配时，AI 会自动启用这个技能。你什么都不用做。

# 你说： 帮我检查一下这个 Python 文件的代码质量 # AI 自动识别并启用"代码审查"技能 # 然后按照技能中定义的审查标准检查
⌨️ 手动触发

斜杠命令

在对话框中输入 /技能名 来手动启用某个技能。

# 输入斜杠命令： /code-review /create-test /generate-docs # 就像微信输入 / 调出小程序或快捷指令

创建自定义技能

你也可以创建自己的技能！有两种方式：让 AI 帮你生成，或自己手写。

方式一：让 AI 帮你创建（推荐 👍）

1

描述你想要的技能

在对话框中使用 /create-skill 命令，然后描述技能的功能：

/create-skill 创建一个代码审查技能， 检查变量命名是否规范、 注释是否完整、 异常处理是否正确
2

AI 引导你完善

AI 会问你几个问题来完善技能的细节：触发条件是什么？检查标准有哪些？输出格式是什么样的？然后自动生成 SKILL.md 文件。

3

保存即可使用

技能文件保存后，重启 IDE 或等待自动加载即可生效。之后就可以用 /技能名 来调用了。

方式二：手动编写 SKILL.md

如果你想自己写，SKILL.md 的格式很简单——就是一个普通的 Markdown 文件，开头有几行配置信息：

--- name: code-review description: 对代码进行专业审查，检查规范性和潜在问题 --- # 代码审查技能 ## 审查标准 当用户请求代码审查时，请按以下标准检查： ### 1. 命名规范 - 变量名使用 snake_case（Python） - 函数名应具有描述性 - 避免单字母变量名（除循环变量外） ### 2. 代码质量 - 函数长度不超过 50 行 - 必须有 docstring 说明 - 异常处理不能使用裸露的 except ### 3. 输出格式 对每个发现的问题，给出： - 📍 位置（文件名 + 行号） - ❌ 问题描述 - ✅ 建议修改
💡

文件开头的 --- 之间是"配置信息"，包含技能名称（name）和描述（description）。--- 之下是具体指令，用 Markdown 格式写——越详细越好，AI 会严格按照你写的内容执行。

📌 适合文科生的技能idea

🧪

自动生成测试

分析你写的函数，自动生成测试代码——帮你检查函数写得对不对。

适合：完成编程作业后自我检查

📖

代码注释生成

自动给你的代码添加详细的中文注释，每一行都解释清楚。

适合：提交作业前确保注释完整

🔄

代码风格统一

按照 PEP 8 规范统一代码风格：修正缩进、换行、命名。

适合：代码写完后"整理卷面"

📊

数据分析模板

给一份 CSV 文件，自动生成完整的探索性数据分析(EDA)代码。

适合：数据分析课程作业

🛡️

错误诊断

当代码报错时，自动分析错误信息，用大白话解释错误原因和修复方法。

适合：看不懂英文报错信息的初学者

📝

作业报告生成

根据代码和运行结果，自动生成格式规范的作业报告（Markdown 格式）。

适合：需要连代码带分析报告一起交的作业

自定义指令

指令是比技能更简单的"规则"——不需要创建文件，直接在设置里输入文字就行。

📋 技能 vs 指令——到底用哪个？

对比维度技能 Skills指令 Instructions
复杂度复杂（需要创建文件夹）简单（设置里输入文字）
触发方式自动/手动（ /命令）始终自动生效
适合场景特定任务的专业处理通用的行为规范
存储位置skills/ 文件夹设置面板中
通俗比喻专业培训课公司的员工手册
💡

初学者建议：先在设置里配几条指令（"注释用中文""遵循 PEP 8"），等熟练了再创建技能。

💡 推荐给初学者的指令

# 推荐的通用指令（复制粘贴即可使用） 1. 所有代码注释和 print 输出使用中文 2. 变量名和函数名使用英文 snake_case 风格 3. 每个函数必须包含中文 docstring 4. 遵循 PEP 8 编码规范 5. 我是编程初学者，解释代码时用最简单的语言 6. 遇到报错时，先用中文解释错误原因，再给出修复方案 7. 字符串格式化优先使用 f-string 8. 代码文件开头添加功能说明注释

📂 项目级规则文件

除了在设置面板写指令，还可以在项目根目录创建 .lingma/rules/ 文件夹，放入 Markdown 规则文件。详见规则系统 →

📌

始终生效

无论何时对话，这条规则都会被 AI 参考。适合代码风格规范。

🤖

模型自动决策

AI 根据对话内容判断是否需要参考该规则。适合某类任务的专属规范。

📁

按文件匹配

只在编辑特定文件类型时生效，如只在 *.py 文件中应用 Python 规范。
