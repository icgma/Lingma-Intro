编程智能体（Agent）

通义灵码的编程智能体可以自主规划、自主执行，端到端地完成编码任务。

🤖 核心概念

什么是编程智能体？

编程智能体不同于简单的代码补全或问答。它具备自主决策能力——你只需用自然语言描述任务目标，智能体会自动拆解为具体步骤，检索代码、编辑文件、运行命令，一步步完成整个任务。

🔧 智能体核心能力

📋

任务拆解

将复杂需求自动拆分为可执行的小步骤，按顺序逐步完成。支持中途调整和回滚。

🔍

环境感知

自动感知项目的框架、技术栈、文件结构和代码上下文，无需你手动提供背景信息。

📝

多文件编辑

可以同时修改多个代码文件，进行工程级别的变更。每次修改都会展示给你确认。

💻

终端命令

自主编写和运行终端命令，如安装依赖、运行测试、启动服务等。

🔌

MCP 工具调用

智能体可以调用你配置的 MCP 工具，访问外部资源。了解 MCP →

🧠

长期记忆

记住你的编码偏好和工作习惯，每次对话的经验会积累到后续交互中。

🚀 如何使用智能体

1

打开 Quest 模式

按 Ctrl+E 进入 Quest 模式，这是智能体的主要工作界面。也可以在对话框左下角切换到智能体模式。

2

描述你的任务

用自然语言描述你想要完成的任务。描述越清晰，智能体的执行越精准。

# ✅ 好的描述 帮我创建一个 Flask Web 应用，包含用户注册和登录功能， 使用 SQLite 数据库，密码用 bcrypt 加密。 # ❌ 过于模糊的描述 帮我写一个网站。
3

审查并确认

智能体会展示它的执行计划和每一步的代码变更。你可以：

  • 接受：确认变更并继续下一步
  • 修改：提出调整意见让智能体重新执行
  • 回滚：撤销当前步骤的变更

技能（Skills）

技能是将专业知识打包成可复用功能的机制。每个技能让 AI 具备特定领域的专业能力。

📦 什么是技能？

你可以把技能想象成给 AI 装上的"专业插件"。每个技能包含一组指令和参考资料，告诉 AI 如何处理某类特定任务。

例如，一个"代码审查"技能会告诉 AI 具体的审查标准、关注点和输出格式。

📁 技能的结构

my-skill/ ├── SKILL.md # 必需：主指令文件 ├── scripts/ # 可选：辅助脚本 ├── examples/ # 可选：参考示例 └── resources/ # 可选：其他资源

⚡ 技能的两种触发方式

🤖 自动触发

AI 智能识别

直接描述需求，AI 会根据技能的描述自动判断是否调用。

# 你说： 帮我分析这个日志文件里的错误 # AI 自动调用 log-analyzer 技能
⌨️ 手动触发

斜杠命令

输入 /技能名 手动触发特定技能。

# 手动调用技能： /code-review /log-analyzer /create-test

📍 技能的存放位置

👤

用户级技能

存放在 ~/.lingma/skills/ 目录下。适用于你个人所有项目的通用技能。

~/.lingma/skills/my-skill/SKILL.md
📂

项目级技能

存放在项目根目录的 .lingma/skills/ 下。适用于团队共享的项目专属技能。

.lingma/skills/my-skill/SKILL.md

创建自定义技能

两种方式创建技能：使用内置助手自动生成，或手动编写 SKILL.md。

方式一：自动创建（推荐）

1

使用 create-skill 命令

在 AI 对话框中输入：

/create-skill 创建一个代码审查技能， 检查变量命名规范、注释完整性和异常处理
2

交互式引导

AI 会引导你完善技能的描述、触发条件和执行逻辑，自动生成 SKILL.md 模板。

3

保存并重启

技能文件保存到对应目录后，重启 IDE 即可生效。

方式二：手动编写 SKILL.md

--- name: code-review description: 对代码进行专业审查，检查规范性和潜在问题 --- # 代码审查技能 ## 审查标准 当用户请求代码审查时，请按以下标准检查： ### 1. 命名规范 - 变量名使用 snake_case（Python） - 函数名应具有描述性 - 避免单字母变量名（除循环变量外） ### 2. 代码质量 - 函数长度不超过 50 行 - 必须有 docstring 说明 - 异常处理不能使用裸露的 except ### 3. 输出格式 对每个问题，给出： - 📍 位置（文件名 + 行号） - ❌ 问题描述 - ✅ 建议修改
💡

SKILL.md 使用 YAML 前置元数据（frontmatter）定义 name 和 description，下方用 Markdown 编写具体指令，越详细越好。

📌 实用技能示例

🧪

自动生成测试

分析函数签名和逻辑，自动生成 pytest 测试用例，覆盖边界条件。

📖

API 文档生成

读取 Python 模块结构，生成 Sphinx/MkDocs 风格的 API 文档。

🔄

代码重构

按照 SOLID 原则和设计模式，对选中的代码块提出重构建议。

📊

数据分析模板

用 pandas + matplotlib 快速生成数据探索性分析(EDA) 脚本。

🛡️

安全审计

检查代码中的常见安全漏洞，如 SQL 注入、XSS、硬编码密钥。

📝

Git 提交规范

根据代码变更自动生成 Conventional Commits 规范的提交信息。

自定义指令

指令是比技能更轻量的自定义方式，用于为 AI 设定特定的行为规范。

📋 技能 vs 指令 — 什么区别？

维度技能 (Skills)指令 (Instructions)
复杂度复杂，有文件结构轻量，一段文字
触发方式自动 / 手动 (/命令)始终生效
适用场景特定任务流程通用行为规范
存储位置skills/ 文件夹设置面板

💡 指令示例

# 示例指令（在设置面板「规则」中配置）： 1. 所有 Python 代码遵循 PEP 8 风格 2. 变量名和函数名用英文，注释用中文 3. 每个函数必须包含 docstring 4. 优先使用 f-string 而非 format() 5. 避免使用 global 变量 6. 错误处理使用具体异常类型

📂 项目专属规则文件

除了在设置面板配置指令，还可以在项目根目录创建 .lingma/rules/ 文件夹，放入 Markdown 文件作为项目专属规则。规则支持以下生效模式：

🤖

模型自动决策

AI 根据当前对话内容，自动判断是否引用该规则。

📌

始终生效

无论何种对话场景，该规则始终被包含在 AI 的上下文中。

📁

按文件匹配

只在编辑特定类型文件（如 *.py）时生效。
