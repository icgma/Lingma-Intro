MCP 服务 详解

给 AI 装上"外挂"——让它能搜索网页、查数据库、读文档，不再局限于聊天。

📖 开始阅读 ← 返回设置

什么是 MCP？

MCP（Model Context Protocol）是一种让 AI 连接外部工具的标准协议——听着复杂，其实很好理解。

🧠 通俗理解

一个比喻就够了

你可以把 AI 想象成一个聪明但被关在房间里的助手。它很会写代码，但看不到房间外面的世界——不能上网搜索、不能查数据库、不能读你电脑上的文档。

MCP 就是给这个房间开了几扇窗户。通过不同的 MCP 服务（窗户），AI 可以：

  • 🔍 通过 Google Search 窗户搜索最新信息
  • 📚 通过 Fetch 窗户阅读网页内容
  • 🗃️ 通过 Database 窗户查询数据库
  • 📝 通过 Notion 窗户读写你的笔记

🔧 两种通信方式

MCP 工具和 AI 之间有两种"对话方式"，了解一下即可，不影响使用：

💻

STDIO 模式常见

工具在你的电脑上本地运行。就像在电脑里安装了一个软件，打开就用。

特点：速度快、隐私好（数据不出电脑）、大部分工具用这种

🌐

SSE 模式

工具在远程服务器上运行，通过网络连接。就像用网页版的在线工具。

特点：无需安装、随时可用、适合团队共享的服务

如何安装 MCP 服务

两种安装方式：去"MCP 广场"一键安装（推荐），或手动配置。

方式一：MCP 广场安装（推荐 👍）

1

打开 MCP 设置

在灵码设置面板中，找到左侧的 "MCP" 选项，点击进入。

2

浏览 MCP 广场

点击 "MCP 广场" 按钮，就像进了一个应用商店：里面列出了所有可用的 MCP 工具，有描述、分类和安装按钮。

3

一键安装

找到你需要的工具，点击 "安装" 按钮。安装完成后，灵码会自动加载这个工具，无需重启。

4

在对话中使用

安装完成后，AI 就可以使用这个工具了。你不需要额外操作——当 AI 判断需要用到某个工具时，会自动调用。

# 安装了 Google Search MCP 后： 请帮我查一下 Python pandas 库的最新版本是多少 # AI 会自动搜索网页并告诉你答案 # 你会看到对话中出现"正在使用 Google Search..."的提示

方式二：手动配置（进阶）

如果 MCP 广场里没有你需要的工具（很少见），可以手动添加。在设置面板的 MCP 页面，点击"添加自定义 MCP"，填入配置信息：

{ "mcpServers": { "fetch": { "command": "uvx", "args": ["mcp-fetch"] } } }
📝

手动配置需要一定的技术基础。初学者使用 MCP 广场一键安装就够了，无需手动配置。

MCP 广场热门工具

以下是对编程初学者和文科学生最有用的几个 MCP 工具。

🔍

Google Search

让 AI 能够搜索谷歌，获取最新信息。再也不用担心 AI 的知识过时了。

实用场景：查技术文档、找错误解决方案、了解最新库版本

📄

Fetch（网页读取）

让 AI 能够读取任意网页的内容。你给它一个链接，它帮你提取关键信息。

实用场景：阅读 API 文档、获取数据源页面内容、分析网页结构

🧠

Sequential Thinking

让 AI 按步骤一步步思考，减少"拍脑袋"式的错误回答。让推理更可靠。

实用场景：解决复杂逻辑问题、编写多步骤算法

🐙

GitHub

让 AI 直接访问你的 GitHub 仓库，查看代码、创建 Issue、管理分支。

实用场景：团队协作、代码版本管理、查看作业提交

🗃️

SQLite / MySQL

让 AI 直接查询数据库，帮你写 SQL 语句并执行，查看查询结果。

实用场景：数据分析作业、学习数据库课程

📐

Playwright（网页自动化）

让 AI 控制浏览器自动操作网页：点击按钮、填写表单、截图。

实用场景：测试你做的网页、爬取公开数据

💡

入门推荐：先安装 Google Search 和 Fetch 两个工具，这能覆盖大部分课程需求。等需要时再安装其他工具。

文科生实用场景

以下是你在课程中可能用到 MCP 的真实场景。

📚

场景一：查阅技术文档

做作业时，不知道某个函数怎么用。让 AI 帮你去官方文档网站查。

# 安装了 Fetch MCP 后 帮我看一下 pandas 官方文档中 DataFrame.groupby() 函数的用法， 并给我一个简单的中文示例。

效果：AI 会访问 pandas 官方文档网站，读取相关内容，用中文给你解释并提供示例代码。

🐛

场景二：搜索错误解决方案

代码报错了看不懂错误信息。让 AI 搜索一下这个错误的典型解决办法。

# 安装了 Google Search MCP 后 运行代码时出现这个错误： ModuleNotFoundError: No module named 'jieba' 帮我搜一下怎么解决。

效果：AI 会搜索这个错误信息，告诉你需要先执行 pip install jieba 来安装这个库。

🎨

场景三：设计稿转网页

你在 Figma 或 Canva 上做了一个网页设计稿，想转成真正的网页代码。

# 安装了 Playwright MCP 后 帮我打开这个 Figma 链接，查看设计稿， 然后按照设计稿的布局和颜色写出 HTML + CSS 代码。
📊

场景四：分析数据库中的数据

有一个 SQLite 数据库文件，想查询里面的数据但不太会写 SQL。

# 安装了 SQLite MCP 后 帮我查看 news.db 数据库里有哪些表， 然后查出 2025 年发布的所有新闻， 按发布日期排序。

效果：AI 会自动连接数据库，列出所有表的结构，编写并执行 SQL 查询语句，返回结果。

常见问题

关于 MCP 服务，初学者最常问的几个问题。

❓

MCP 是免费的吗？

灵码自带的 MCP 广场中的工具大部分是免费开源的。但部分工具可能需要额外的 API Key（如 Google Search 需要配置搜索 API）。课堂上老师会指导你完成配置。

❓

不安装 MCP 影响使用吗？

完全不影响。MCP 是锦上添花的高级功能。没有 MCP，AI 依然可以帮你写代码、解释代码、回答问题。只是无法访问外部资源（如搜索网页、查数据库）。

❓

安装了 MCP 但 AI 没有使用？

AI 会自己判断是否需要调用 MCP 工具。如果你想让 AI 主动使用，可以在对话中明确提示，例如"请使用搜索工具查一下"或"帮我用 Fetch 读取这个网页"。

❓

MCP 工具是否安全？

MCP 广场中的工具经过审核，应当安全。使用 STDIO 模式的工具数据不出本机。使用 SSE 模式的工具数据会通过网络传输，注意不要输入个人敏感信息。
